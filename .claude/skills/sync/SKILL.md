# Sync — Синхронизация и публикация

Скилл управляет ветками: каждый пользователь работает в своей ветке, публикует в main через визард.

## Определение пользователя

Получи имя из git:
```bash
git config user.name
```

Нормализуй в имя ветки: lowercase, пробелы → дефисы, убрать спецсимволы.
Пример: `Valery Sayapin` → `valery-sayapin`

## Режим 1: Начало работы

Вызывай когда пользователь начинает работу или в начале сессии.

### Шаги

1. Определи пользователя (git user.name)
2. Проверь есть ли ветка пользователя (`git branch --list <имя>`)
3. Если ветки нет — создай от свежего main:
   ```bash
   git checkout main && git pull origin main && git checkout -b <имя>
   ```
4. Если ветка есть — переключись и подтяни main:
   ```bash
   git checkout <имя> && git pull origin main --no-edit
   ```
5. Если при подтягивании main возникли конфликты — запусти визард конфликтов (см. ниже)

### Вывод

```
🔄 ═══════════════════════════════════════
   SYNC · НАЧАЛО РАБОТЫ
═══════════════════════════════════════
👤 Пользователь ................ Valery Sayapin
🌿 Ветка ....................... valery-sayapin
📥 Main ........................ ✅ Синхронизирован
═══════════════════════════════════════
   Готово! Работай. Когда закончишь — скажи «опубликуй»
═══════════════════════════════════════
```

## Режим 2: Публикация в main

Вызывай когда пользователь говорит: «опубликуй», «мерж», «в main», «готово, публикуй», «sync publish».

### Шаги

1. Проверь что все изменения закоммичены. Если нет — предложи закоммитить
2. Переключись на main и подтяни свежую версию:
   ```bash
   git checkout main && git pull origin main
   ```
3. Мержи ветку пользователя:
   ```bash
   git merge <ветка> --no-edit
   ```
4. Если конфликтов нет:
   ```bash
   git push origin main
   ```
5. Если конфликты — запусти визард конфликтов
6. После успешного мержа — переключись обратно на ветку пользователя и подтяни main:
   ```bash
   git checkout <ветка> && git pull origin main --no-edit
   ```

### Вывод при успехе

```
🚀 ═══════════════════════════════════════
   SYNC · ПУБЛИКАЦИЯ
═══════════════════════════════════════
👤 Пользователь ................ Valery Sayapin
🌿 Ветка ....................... valery-sayapin → main
📦 Коммиты ..................... 3 коммита влиты
🔄 Push ........................ ✅ Отправлено
═══════════════════════════════════════
   Опубликовано в main!
═══════════════════════════════════════
```

## Визард конфликтов

Когда merge даёт конфликты — НЕ решай молча. Проведи пользователя через каждый файл.

### Для каждого конфликтного файла:

1. Покажи понятное описание:
```
⚠️ Конфликт в файле: src/components/ui/Form.astro

📄 Версия из main (коллега, 25 фев 14:30):
   Изменён стиль кнопки — добавлен hover-эффект

📄 Твоя версия (26 фев 10:15):
   Добавлено новое поле «телефон»

Что делаем?
  1) Оставить версию из main (коллеги)
  2) Оставить твою версию
  3) Объединить оба изменения (я попробую автоматически)
```

2. Дождись ответа пользователя
3. Примени выбор
4. Переходи к следующему файлу

### Определение дат и авторов

Для показа информации о конфликтующих изменениях:
```bash
git log main --oneline -1 -- <файл>
git log <ветка> --oneline -1 -- <файл>
```

### После разрешения всех конфликтов

```bash
git add . && git commit --no-edit
```

Продолжи с push как в обычном режиме публикации.

## Важно

- НИКОГДА не делай force push
- НИКОГДА не удаляй чужие ветки
- При сомнениях — спрашивай пользователя
- Ветка пользователя НЕ удаляется после мержа — она постоянная
